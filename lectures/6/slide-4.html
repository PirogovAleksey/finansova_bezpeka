CVSS Base Score

Складові оцінки

🌐 Attack Vector — Network / Adjacent / Local / Physical
🧩 Attack Complexity — Low (тривіально) / High (потрібні умови)
🔑 Privileges Required — None / Low / High
👤 User Interaction — None (автоматично) / Required (клік)
💥 CIA Impact — Confidentiality, Integrity, Availability: None/Low/High

SQLi в API банку: AV:N, AC:L, PR:N, UI:N, C:H, I:H, A:H = CVSS 9.8 (Critical).

Vulnerability Lifecycle

Від виявлення до закриття

1
Виявлення (Discovery)
Сканер, пентест, bug bounty, code review, інцидент
2
Тріаж (Triage)
Підтвердження, CVSS-оцінка, визначення пріоритету, присвоєння власника
3
Виправлення (Remediation)
Code fix, virtual patching (WAF), оновлення залежностей, конфігураційні зміни
4
Верифікація (Verification)
Re-testing: PoC більше не працює, regression test пройдено
5
Закриття та метрики (Closure & Metrics)
Закриття тікету, розрахунок MTTR, lessons learned, оновлення playbook

Середній MTTR у фінансовому секторі: Critical 3 дні, High 15 днів, Medium 45 днів. Мета: зменшити вдвічі.

Responsible Disclosure Process

Етичне розкриття вразливостей

1
Виявлення
Дослідник знаходить вразливість у рамках scope (або випадково)
2
Звіт вендору
Конфіденційне повідомлення через security@, bug bounty платформу або security.txt
3
Координація
Вендор підтверджує, оцінює CVSS, узгоджує таймлайн виправлення (зазвичай 90 днів)
4
Виправлення
Вендор випускає патч, CVE присвоюється через MITRE/CNA
5
Публікація
Публічне розкриття після патчу: advisory, блог-пост, CVE деталі. Винагорода дослідника

ISO 29147 (disclosure) + ISO 30111 (handling) — міжнародні стандарти процесу розкриття вразливостей.

🛡️

Ключові висновки

1️⃣ CVSS — стандарт оцінки: 9.0+ = Critical, fix за 24-72 год
2️⃣ Пріоритизація — CVSS + бізнес-контекст, не лише технічна оцінка
3️⃣ MTTR — ключова метрика, зменшуйте час від discovery до fix
4️⃣ Bug Bounty — безперервне доповнення до періодичних пентестів
