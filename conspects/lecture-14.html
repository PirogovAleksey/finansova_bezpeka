Ризик-менеджмент та SOC

⏱ 2 годКонспект

14.1 Фреймворки ризик-менеджменту

Управління кіберризиками — це систематичний процес ідентифікації, оцінки та обробки ризиків, що загрожують інформаційним активам організації. Для фінансових установ ризик-менеджмент є не просто best practice, а регуляторною вимогою (НБУ, Basel III, DORA).

Ключові фреймворки:

  • ISO 27005 — міжнародний стандарт управління ризиками інформаційної безпеки. Визначає процес: context establishment → risk identification → risk analysis → risk evaluation → risk treatment → monitoring & review.
  • NIST Risk Management Framework (RMF) — SP 800-37: categorize → select → implement → assess → authorize → monitor. Широко використовується у фінансовому секторі США та глобально.
  • NIST Cybersecurity Framework (CSF) 2.0 — 6 функцій: Govern, Identify, Protect, Detect, Respond, Recover. Гнучкий фреймворк для організацій будь-якого розміру.
  • FAIR (Factor Analysis of Information Risk) — кількісна модель для оцінки кіберризиків у фінансових термінах (долари/гривні). Дозволяє порівнювати ризики з іншими бізнес-ризиками.

Risk Appetite та Risk Tolerance:

  • Risk appetite — загальний рівень ризику, який організація готова прийняти для досягнення бізнес-цілей. Визначається правлінням/наглядовою радою.
  • Risk tolerance — допустимі відхилення від risk appetite для конкретних ризиків.
  • Risk register — реєстр усіх ідентифікованих ризиків з оцінкою ймовірності, впливу, поточних контролів та планів обробки.

14.2 Оцінка та обробка ризиків

Оцінка ризиків визначає пріоритети для інвестицій у безпеку. Існують два основні підходи:

Якісна оцінка (Qualitative):

  • Використовує шкали (Low/Medium/High/Critical) для ймовірності та впливу.
  • Risk matrix (heat map): ймовірність x вплив = рівень ризику.
  • Переваги: простота, швидкість, не вимагає точних даних. Недоліки: суб'єктивність, складно порівнювати з бізнес-ризиками.

Кількісна оцінка (Quantitative):

  • ALE (Annualized Loss Expectancy) = SLE (Single Loss Expectancy) x ARO (Annualized Rate of Occurrence).
  • Приклад: data breach (SLE = $5M) x ймовірність 10% на рік (ARO = 0.1) = ALE = $500K/рік.
  • FAIR model дозволяє Monte Carlo симуляцію для розрахунку розподілу можливих збитків.
  • Переваги: об'єктивність, фінансова мова для правління. Недоліки: вимагає даних, складніша.

Risk Treatment (обробка ризиків):

  • Mitigate (зменшити) — впровадити контролі для зниження ймовірності або впливу. Найпоширеніший варіант.
  • Transfer (передати) — кіберстрахування, аутсорсинг безпеки (MSSP).
  • Accept (прийняти) — ризик в межах appetite, вартість mitigation перевищує потенційні збитки.
  • Avoid (уникнути) — відмовитися від діяльності, що створює ризик (наприклад, не зберігати певні дані).

Residual risk — ризик, що залишається після застосування контролів. Повинен бути в межах risk appetite. Якщо ні — потрібні додаткові заходи або acceptance від керівництва.

14.3 Архітектура та функціонування SOC

Security Operations Center (SOC) — це централізований підрозділ, що забезпечує безперервний моніторинг, виявлення та реагування на кіберзагрози. Для великих фінансових установ SOC працює 24/7/365.

SOC Tiers (рівні аналітиків):

  • Tier 1 (Alert Triage) — моніторинг алертів, початкова класифікація, ескалація. Використовує runbooks для стандартних процедур.
  • Tier 2 (Incident Analysis) — глибокий аналіз ескальованих інцидентів, кореляція даних з різних джерел, визначення scope та impact.
  • Tier 3 (Threat Hunting / Advanced) — проактивний пошук загроз, аналіз malware, розробка detection rules, threat intelligence.
  • SOC Manager — координація команди, метрики, reporting, взаємодія з бізнесом та регуляторами.

Ключові технології SOC:

  • SIEM (Security Information and Event Management) — централізований збір та кореляція логів з усіх джерел: Splunk, Microsoft Sentinel, Elastic SIEM. Правила кореляції, dashboards, alerting.
  • EDR (Endpoint Detection and Response) — моніторинг endpoints у реальному часі: CrowdStrike Falcon, Microsoft Defender for Endpoint, SentinelOne. Виявлення malware, lateral movement, fileless attacks.
  • SOAR (Security Orchestration, Automation and Response) — автоматизація реагування: Palo Alto XSOAR, Splunk SOAR. Автоматична ізоляція хостів, блокування IP, збагачення алертів.
  • Threat Intelligence Platform (TIP) — агрегація IOC з зовнішніх та внутрішніх джерел, автоматичне збагачення алертів контекстом.

SOC Models: In-house SOC (повний контроль, дорого), MSSP/MDR (Managed Security Service Provider — аутсорсинг, дешевше), Hybrid (критичні функції in-house, решта — MSSP).

14.4 Метрики та зрілість кібербезпеки

Метрики дозволяють вимірювати ефективність кібербезпеки, обґрунтовувати інвестиції та демонструвати прогрес керівництву та регуляторам.

Ключові метрики (KPI/KRI):

  • MTTD (Mean Time to Detect) — середній час від початку інциденту до його виявлення. Ціль: <24 години (industry average: 204 дні).
  • MTTR (Mean Time to Respond) — середній час від виявлення до containment. Ціль: <4 години для critical.
  • Alert-to-incident ratio — скільки алертів стають реальними інцидентами. Показує якість detection rules.
  • False positive rate — відсоток хибних спрацювань. Високий FP = alert fatigue = пропущені реальні загрози.
  • Patch compliance — відсоток систем з актуальними патчами. Ціль: >95% протягом SLA.
  • Phishing click rate — результати phishing-симуляцій. Ціль: <5%.

SOC Maturity Model (CMM):

  • Level 1 — Initial: reactive, ad hoc процеси, мінімальний моніторинг.
  • Level 2 — Managed: базовий SIEM, визначені процеси, runbooks для типових інцидентів.
  • Level 3 — Defined: повноцінний SOC, threat hunting, автоматизація через SOAR, threat intelligence.
  • Level 4 — Quantitatively Managed: метрики-driven, predictive analytics, continuous improvement.
  • Level 5 — Optimizing: AI-driven SOC, автономне реагування, інноваційні підходи.

Reporting для керівництва: CISO dashboard з ключовими KRI, risk posture, compliance status, incident trends. Мова бізнесу: не «blocked 10K attacks», а «зменшили ризик втрати на $2M через покращення detection».

Підсумок

Ризик-менеджмент визначає стратегію кібербезпеки: фреймворки (ISO 27005, NIST, FAIR) допомагають систематично ідентифікувати, оцінювати та обробляти ризики. Кількісна оцінка (ALE, FAIR) дозволяє обґрунтувати інвестиції фінансовою мовою.

SOC є оперативним ядром кібербезпеки: 24/7 моніторинг через SIEM/EDR, автоматизація реагування через SOAR, проактивний threat hunting. Метрики (MTTD, MTTR, false positive rate) та SOC maturity model забезпечують вимірюваний прогрес.

У наступній лекції ми розглянемо національну стратегію фінансової кібербезпеки: роль НБУ та CERT-UA, міжнародні стандарти NIS2 та DORA, досвід кібербезпеки в умовах війни та стратегічні пріоритети розвитку.
