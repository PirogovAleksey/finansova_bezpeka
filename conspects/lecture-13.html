Кібергігієна та захист кінцевого користувача

⏱ 2 год Конспект

13.1 Паролі та управління акаунтами

Слабкі та повторювані паролі залишаються одним із найпоширеніших векторів компрометації. За даними Verizon DBIR, понад 80% зломів, пов'язаних з хакерством, використовують вкрадені або слабкі паролі. Середній користувач має 100+ онлайн-акаунтів, що робить ручне управління унікальними паролями неможливим.

Password Managers:

  • Генерація та зберігання унікальних складних паролів для кожного сервісу (Bitwarden, 1Password, KeePass).
  • Master password — єдиний пароль, який потрібно запам'ятати. Повинен бути довгим (16+ символів) та унікальним.
  • Автозаповнення запобігає введенню паролю на фішинговому сайті: менеджер перевіряє домен.

Multi-Factor Authentication (MFA):

  • SMS OTP — найслабший фактор: SIM-swap, SS7-перехоплення. Краще, ніж нічого, але не для критичних акаунтів.
  • TOTP (Google Authenticator, Authy) — одноразовий код, генерований додатком. Значно безпечніший за SMS.
  • FIDO2/WebAuthn (Passkeys) — криптографічна автентифікація прив'язана до пристрою та домену. Phishing-resistant за дизайном. Підтримується Google, Apple, Microsoft.
  • Hardware keys (YubiKey) — фізичний пристрій для FIDO2. Золотий стандарт для високоцінних акаунтів.

Breach monitoring — сервіси типу Have I Been Pwned перевіряють, чи ваші облікові дані з'явилися у відомих витоках. Password managers (1Password, Bitwarden) мають вбудовану перевірку Watchtower/Vault Health.

13.2 Захист персональних даних

Персональні дані є цінним активом для зловмисників: від identity theft до відкриття фінансових рахунків на чуже ім'я. Захист приватності — це не лише право, а й фінансова безпека.

GDPR та принципи захисту даних:

  • Data minimization — збирайте лише ті дані, які дійсно потрібні. Не зберігайте «про всяк випадок».
  • Purpose limitation — дані використовуються лише для мети, заради якої зібрані.
  • Storage limitation — видаляйте дані, коли вони більше не потрібні.
  • Право на забуття — клієнт має право вимагати видалення своїх персональних даних.
  • Data breach notification — повідомлення регулятора протягом 72 годин, постраждалих — «без невиправданої затримки».

Практичні кроки для приватності:

  • Обмежте публічну інформацію в соцмережах (LinkedIn, Facebook) — це OSINT для spear phishing.
  • Використовуйте різні email-адреси для різних категорій (фінанси, соцмережі, shopping).
  • Перевіряйте дозволи мобільних додатків: камера, мікрофон, локація — лише коли дійсно потрібно.
  • Використовуйте приватний режим браузера, блокувальники трекерів (uBlock Origin), DNS-фільтрацію.

13.3 Безпека пристроїв та мереж

Кінцеві пристрої (ноутбуки, смартфони, планшети) та домашні мережі є першою лінією оборони для більшості користувачів. Компрометація пристрою = доступ до банківських додатків, email, корпоративних систем.

Endpoint Security:

  • Оновлення — автоматичне оновлення ОС та додатків. 60% успішних атак використовують відомі, але невиправлені вразливості.
  • Антивірус / EDR — Windows Defender достатній для більшості, корпоративні рішення (CrowdStrike, SentinelOne) для бізнесу.
  • Full Disk Encryption — BitLocker (Windows), FileVault (macOS), LUKS (Linux). При крадіжці пристрою дані захищені.
  • Screen lock — автоматичне блокування через 1-5 хвилин, PIN/біометрія для розблокування.

Безпека мережі:

  • Домашній Wi-Fi — WPA3 (або WPA2), складний пароль, зміна default admin credentials роутера, оновлення firmware.
  • Публічний Wi-Fi — НІКОЛИ не вводьте фінансові дані у публічних мережах без VPN. MITM-атаки тривіальні в кафе та аеропортах.
  • VPN — шифрований тунель для публічних мереж. Надійні провайдери: WireGuard-based рішення, корпоративний VPN для роботи.
  • IoT — розумні пристрої (камери, термостати) в окрему мережу (VLAN/guest network), регулярні оновлення firmware.

13.4 Фінансова безпека онлайн

Онлайн-фінанси вимагають підвищеної пильності: від безпечних платежів до розпізнавання інвестиційних скамів.

Безпечні онлайн-платежі:

  • Використовуйте віртуальні картки для онлайн-покупок (одноразові номери, ліміти).
  • Перевіряйте HTTPS та легітимність сайту перед введенням даних картки.
  • Apple Pay / Google Pay безпечніші за введення номера картки (токенізація).
  • Увімкніть push-сповіщення про транзакції для миттєвого виявлення несанкціонованих операцій.

Розпізнавання шахрайства:

  • Investment scams — обіцянки гарантованого прибутку 100%+, «ексклюзивна можливість», тиск на швидке рішення. Якщо звучить занадто добре — це скам.
  • Romance scams — побудова онлайн-стосунків із подальшим запитом на гроші. Особливо поширені на dating-платформах.
  • Fake apps — підроблені банківські додатки в App Store / Google Play. Завантажуйте лише з офіційних посилань банку.
  • Правило: банк НІКОЛИ не просить PIN, CVV, повний номер картки або OTP-код по телефону або email.

Моніторинг та реагування: підключіть SMS/push-сповіщення про всі операції, регулярно перевіряйте виписки, використовуйте сервіси моніторингу кредитної історії, негайно блокуйте картку при підозрілій транзакції через мобільний додаток.

Підсумок

Кібергігієна — це щоденні практики, що значно зменшують ризик кібератак. Password manager + MFA (бажано FIDO2) закривають найпоширеніший вектор — компрометацію облікових даних. Захист персональних даних та обмеження цифрового сліду ускладнюють spear phishing та identity theft.

Оновлення пристроїв, шифрування дисків та обережність у публічних мережах захищають від технічних атак. Для фінансової безпеки: віртуальні картки, push-сповіщення та здоровий скептицизм до «вигідних пропозицій» є обов'язковими.

У наступній лекції ми розглянемо ризик-менеджмент та SOC: управління кіберризиками, побудову та функціонування центрів безпеки (Security Operations Center) у фінансових установах.
